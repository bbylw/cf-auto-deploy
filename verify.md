# 🧪 Verify Capability

Verify that a deployed Cloudflare Worker is functioning correctly. Three verification levels from quick smoke test to deep behavioral validation.

## When to Verify

- After every deploy (smoke test)
- After every iteration (endpoint or behavioral test)
- Before claiming a temporary account (full validation)
- When user asks "does it work?", "test the endpoint", "check the response"

## Prerequisites

- A deployed Worker with known preview URL
- `curl` available (or `Invoke-WebRequest` on Windows PowerShell)
- Optional: `jq` for JSON parsing

## Verification Levels

### Level 1: Smoke Test (Quick)

Run after every deploy — basic "is it alive?" check:

```bash
# HTTP status code
curl -sS -o /dev/null -w "%{http_code}" "<preview-url>"

# Response body (first 500 chars)
curl -sS "<preview-url>" | head -c 500

# Response headers
curl -sSI "<preview-url>"
```

**Windows PowerShell:**
```powershell
$resp = Invoke-WebRequest -Uri "<preview-url>" -UseBasicParsing
$resp.StatusCode
$resp.Content.Substring(0, [Math]::Min(500, $resp.Content.Length))
```

**Pass criteria:**
- HTTP 200 (or expected status code)
- Non-empty response body
- Response time < 5s

### Level 2: Endpoint Tests (Standard)

Test each known endpoint/route:

```bash
# Test root
curl -sS "<preview-url>/"

# Test API endpoints
curl -sS "<preview-url>/api/health"
curl -sS "<preview-url>/api/status"

# Test with different HTTP methods
curl -sS -X POST "<preview-url>/api/data" -d '{"test": true}'

# Test with headers
curl -sS -H "Authorization: Bearer test" "<preview-url>/api/protected"
```

### Level 3: Behavioral Validation (Deep)

Compare actual vs. expected:

```bash
# JSON field check
ACTUAL=$(curl -sS "<preview-url>/api/status")
echo "$ACTUAL" | jq -e '.status == "ok"'

# Text contains check
ACTUAL=$(curl -sS "<preview-url>/")
[[ "$ACTUAL" == *"hello cloudflare"* ]] && echo "PASS" || echo "FAIL"

# Status code check
STATUS=$(curl -sS -o /dev/null -w "%{http_code}" "<preview-url>/nonexistent")
[[ "$STATUS" == "404" ]] && echo "PASS" || echo "FAIL"
```

## Verification Flow

### Step 1: Identify the Preview URL

Get the URL from:
- Most recent `wrangler deploy` output
- `wrangler deployments list` output
- User-provided URL

### Step 2: Discover Endpoints

If endpoints are unknown, discover from source code:
- Grep for route patterns: `request.url`, `url.pathname`, `router.get`, etc.
- Check for framework-specific routing (Hono, itty-router, etc.)
- Default: test `/` (root)

### Step 3: Run Smoke Test

Always start with Level 1 to confirm Worker is reachable.

### Step 4: Run Endpoint Tests

For each discovered endpoint, test and record:

| Endpoint | Method | Expected | Actual | Result |
|----------|--------|----------|--------|--------|
| `/` | GET | 200 + "hello" | 200 + "hello cloudflare" | ✅ PASS |
| `/api/health` | GET | 200 + JSON | 404 | ❌ FAIL |
| `/api/users` | GET | 200 + JSON array | 200 + `[]` | ✅ PASS |

### Step 5: Report Results

```
🧪 Verification Report

Smoke Test:
  ✅ Worker is reachable (HTTP 200, 142ms)

Endpoint Tests:
  ✅ GET /              → 200 "hello cloudflare"
  ❌ GET /api/health    → 404 (expected 200 with JSON)
  ✅ GET /api/users     → 200 []
  ✅ POST /api/data     → 201 {"created": true}

Summary: 3/4 passed (75%)

❌ Failures:
  /api/health returned 404 — route may not be registered
  → Check src/index.ts for the /api/health handler
```

## Automated Test Script

For repeated verification, generate a test script:

### Bash (macOS/Linux)
```bash
#!/bin/bash
# verify.sh — auto-generated by cf-auto-deploy
URL="${1:-https://my-app.xyz.workers.dev}"
PASS=0; FAIL=0

test_endpoint() {
  local method="$1" path="$2" expected="$3"
  local actual=$(curl -sS -o /dev/null -w "%{http_code}" -X "$method" "$URL$path")
  if [[ "$actual" == "$expected" ]]; then
    echo "✅ $method $path → $actual"
    ((PASS++))
  else
    echo "❌ $method $path → $actual (expected $expected)"
    ((FAIL++))
  fi
}

test_endpoint "GET"  "/"            "200"
test_endpoint "GET"  "/api/health"  "200"
test_endpoint "GET"  "/api/users"   "200"
test_endpoint "POST" "/api/data"    "201"

echo ""
echo "Results: $PASS passed, $FAIL failed"
exit $FAIL
```

### PowerShell (Windows)
```powershell
# verify.ps1
$url = "https://my-app.xyz.workers.dev"
$results = @()

$endpoints = @(
    @{ Method = "GET";  Path = "/";           Expected = 200 },
    @{ Method = "GET";  Path = "/api/health"; Expected = 200 },
    @{ Method = "GET";  Path = "/api/users";  Expected = 200 }
)

foreach ($ep in $endpoints) {
    try {
        $resp = Invoke-WebRequest -Uri "$url$($ep.Path)" -Method $ep.Method -UseBasicParsing
        $status = $resp.StatusCode
    } catch {
        $status = $_.Exception.Response.StatusCode.Value__
    }
    $result = if ($status -eq $ep.Expected) { "PASS" } else { "FAIL" }
    $results += [PSCustomObject]@{
        Endpoint = "$($ep.Method) $($ep.Path)"
        Expected = $ep.Expected
        Actual   = $status
        Result   = $result
    }
}

$results | Format-Table
```

## Common Verification Scenarios

### Verify "hello world" change
```
User: I changed "hello world" to "hello cloudflare", check it
→ curl preview URL → grep for "hello cloudflare" → report PASS/FAIL
```

### Verify new API endpoint
```
User: I added /api/status, does it work?
→ curl /api/status → check 200 + valid JSON → report
```

### Verify error handling
```
User: Make sure /api/protected returns 401 without auth
→ curl /api/protected (no auth header) → check 401 → report
→ curl /api/protected (with auth) → check 200 → report
```

### Full regression after iteration
```
User: I refactored the routing, make sure everything still works
→ Run all endpoint tests → compare to previous results → report
```

## Debug with Logs

```bash
npx wrangler tail
```

Streams real-time logs from the Worker — useful for diagnosing 500 errors.

## Error Diagnosis

| Symptom | Likely Cause | Suggested Fix |
|---------|-------------|---------------|
| Connection refused | Worker not deployed | Run `wrangler deploy` |
| 404 on all routes | Wrong URL or Worker name | Check `wrangler deploy` output |
| 500 Internal Error | Code exception | Check `wrangler tail` for logs |
| Timeout | Worker exceeds CPU limit | Optimize code or check infinite loops |
| Wrong response | Code not updated | Confirm `wrangler deploy` succeeded |
| SSL error | Certificate not ready | Wait 30s and retry |
